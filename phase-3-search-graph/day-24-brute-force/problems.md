## 🟢 기초

### 1. 모의고사 — [프로그래머스 #42840](https://school.programmers.co.kr/learn/courses/30/lessons/42840)
- **문제:** 세 수포자가 각자 정해진 규칙으로 답을 찍는다. 정답 배열 `answers`가 주어질 때, 가장 많이 맞힌 사람(들)의 번호를 오름차순으로 반환하라.
- **입력/출력:** `answers=[1,2,3,4,5]` → `[1]`. `answers=[1,3,2,4,2]` → `[1,2,3]`.
- **핵심:** 세 사람의 찍기 패턴을 각각 사이클로 두고, **모든 문제를 전수 비교**하며 점수를 센다. 최고 점수와 같은 사람을 모두 모은다. 완전 탐색의 가장 기초적인 "전수 비교" 형태.
- **힌트:** 각 패턴은 `[1,2,3,4,5]`, `[2,1,2,3,2,4,2,5]`, `[3,3,1,1,2,2,4,4,5,5]` 를 반복. `answers[i] == pattern[i % len(pattern)]` 로 채점.

### 2. Two Sum — [LeetCode #1](https://leetcode.com/problems/two-sum/)
- **문제:** 정수 배열 `nums`와 `target`이 주어질 때, 더해서 `target`이 되는 두 원소의 인덱스를 반환하라.
- **입력/출력:** `nums=[2,7,11,15], target=9` → `[0,1]`.
- **핵심:** 완전 탐색은 **모든 쌍 (i, j)** 를 이중 루프로 확인 `O(n^2)`. 비교용 최적해는 해시맵으로 "target - nums[i]"를 조회하는 `O(n)`([[day-13-hashmap-patterns/concept|Day 13 해시맵]]).
- **힌트:** 완전 탐색으로 먼저 맞힌 뒤, 어떻게 `O(n)`으로 줄이는지 대조하라. "완전 탐색 → 최적화"의 표준 사례.

## 🟡 중급

### 3. Subsets — [LeetCode #78](https://leetcode.com/problems/subsets/)
- **문제:** 서로 다른 정수 배열의 **모든 부분집합**(멱집합, power set)을 반환하라.
- **입력/출력:** `[1,2,3]` → `[[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]` (순서 무관).
- **핵심:** 부분집합 완전 탐색. 원소마다 "넣기/빼기" 이진 선택 → `2^n` 가지. **재귀**(선택-재귀-되돌리기) 또는 **비트마스크**(`for mask in range(1<<n)`) 두 방식 모두 익혀라.
- **힌트:** `n <= 10` 이므로 `2^10 = 1024`, 완전 탐색이 정답. 재귀와 비트마스크 결과가 같은지 확인.

### 4. 소수 찾기 — [프로그래머스 #42839](https://school.programmers.co.kr/learn/courses/30/lessons/42839)
- **문제:** 숫자 카드 문자열 `numbers`(예: `"17"`)의 조각을 이어 붙여 만들 수 있는 모든 수 중 **소수의 개수**를 구하라.
- **입력/출력:** `numbers="17"` → `3` (`7, 17, 71`). `numbers="011"` → `2` (`11, 101`).
- **핵심:** **순열 완전 탐색 + 소수 판정**. 카드 길이 1..n에 대해 `permutations`로 만들 수 있는 모든 수를 `set`에 모으고(중복·앞자리 0 처리), 각 수가 소수인지 검사한다.
- **힌트:** 카드가 최대 7개라 `7! x (1..7 길이) ~= 수천` 수준 → 완전 탐색 가능. 소수 판정은 `2..sqrt(x)` 시분할([[day-05-math/concept|Day 05 수학]]). `int("011")` 로 앞자리 0을 자연 제거.

### 5. Permutations — [LeetCode #46](https://leetcode.com/problems/permutations/)
- **문제:** 서로 다른 정수 배열의 **모든 순열**을 반환하라.
- **입력/출력:** `[1,2,3]` → 6개 순열.
- **핵심:** 순열 완전 탐색. `used[]` 배열로 "아직 안 쓴 원소"를 골라 자리에 놓고 재귀 → `n!`. 비교용으로 `itertools.permutations` 한 줄.
- **힌트:** `n <= 6` 이므로 `6! = 720`. 재귀 골격의 되돌리기(`used[i]=False`, `cur.pop()`)를 정확히 짜는 연습.

### 6. 카펫 — [프로그래머스 #42842](https://school.programmers.co.kr/learn/courses/30/lessons/42842)
- **문제:** 갈색(brown) 격자와 노란색(yellow) 격자 개수가 주어질 때, 카펫의 `[가로, 세로]`를 반환하라. 가로 >= 세로. 테두리는 갈색, 내부는 노란색.
- **입력/출력:** `brown=10, yellow=2` → `[4,3]`. `brown=8, yellow=1` → `[3,3]`.
- **핵심:** **약수 완전 탐색**. 전체 넓이 = `brown + yellow`. 세로를 `1..sqrt(total)` 로 전수하며 가로 = `total // 세로`가 정수인 후보만 검사. 내부 노란 넓이 `(가로-2)*(세로-2) == yellow` 인지 확인.
- **힌트:** 가능한 (가로, 세로) 후보가 약수 개수만큼뿐이라 완전 탐색이 가볍다. 테두리 갈색 = `2*가로 + 2*세로 - 4`.

### 7. Combinations — [LeetCode #77](https://leetcode.com/problems/combinations/)
- **문제:** `1..n` 중에서 `k`개를 고르는 **모든 조합**을 반환하라.
- **입력/출력:** `n=4, k=2` → `[[1,2],[1,3],[1,4],[2,3],[2,4],[3,4]]`.
- **핵심:** 조합 완전 탐색. **시작 인덱스 `start`** 를 넘겨 "이미 지난 원소는 다시 안 고르게" 해 순서 없는 선택을 만든다 → `nCk`. 비교용 `itertools.combinations`.
- **힌트:** 조합은 순열과 달리 `(1,2)`와 `(2,1)`을 같게 본다. `start`로 오름차순만 생성하면 중복이 원천 차단된다.

## 🔴 심화

### 8. Letter Combinations of a Phone Number — [LeetCode #17](https://leetcode.com/problems/letter-combinations-of-a-phone-number/)
- **문제:** `2..9` 숫자 문자열이 주어질 때, 옛날 전화기 자판(2=abc, 3=def, ...)으로 만들 수 있는 **모든 글자 조합**을 반환하라.
- **입력/출력:** `"23"` → `["ad","ae","af","bd","be","bf","cd","ce","cf"]`.
- **핵심:** **곱집합(product) 완전 탐색**. 각 자리의 후보 글자를 축으로 두고 모든 조합 → `∏(각 숫자의 글자 수)`. `itertools.product` 또는 재귀 DFS.
- **힌트:** 숫자당 최대 4글자(7,9), 최대 4자리 → `4^4 = 256`. 빈 문자열이면 빈 리스트 반환하는 예외 처리.

### 9. Combination Sum — [LeetCode #39](https://leetcode.com/problems/combination-sum/)
- **문제:** 서로 다른 양의 정수 `candidates`에서 (같은 수를 여러 번 사용 가능) 합이 `target`이 되는 **모든 조합**을 반환하라.
- **입력/출력:** `candidates=[2,3,6,7], target=7` → `[[2,2,3],[7]]`.
- **핵심:** **재귀 완전 탐색(중복 사용 허용)**. `start` 인덱스부터 원소를 고르되, 같은 원소를 다시 고를 수 있으므로 재귀 시 `start`를 그대로 유지. 남은 합이 음수면 가지치기.
- **힌트:** 순서 없는 조합이라 `start`로 오름차순 유지 → `[2,3,2]` 같은 중복 순열 방지. 이미 백트래킹의 가지치기가 섞인다([[day-27-backtracking/concept|Day 27]] 예고).

### 10. Subsets II — [LeetCode #90](https://leetcode.com/problems/subsets-ii/)
- **문제:** **중복이 있을 수 있는** 정수 배열의 모든 부분집합을 (중복 없이) 반환하라.
- **입력/출력:** `[1,2,2]` → `[[],[1],[1,2],[1,2,2],[2],[2,2]]`.
- **핵심:** 부분집합 완전 탐색 + **중복 처리**. 정렬 후, 같은 값이 연속될 때 "같은 깊이에서 두 번째부터는 건너뛰기"로 중복 부분집합을 차단한다.
- **힌트:** 정렬이 필수. `if i > start and nums[i] == nums[i-1]: continue` 가 중복 제거의 핵심 한 줄. `set`(정렬된 튜플)으로도 유일화 가능하지만 정렬+스킵이 정석.
