## 🟢 문제 1 — 문자열 다루기 기본 (검증 / Validation)

| 항목 | 내용 |
|------|------|
| 출처 | 프로그래머스 (Programmers) |
| 번호 | 12918 |
| 링크 | https://school.programmers.co.kr/learn/courses/30/lessons/12918 |
| 카테고리 | 문자열, 길이 검사, isdigit |
| 난이도 | 🟢 Level 1 |

### 문제 요약

문자열 `s`의 길이가 **4 또는 6**이고, **모든 문자가 숫자**로만 이루어져 있으면 `True`,
아니면 `False`를 반환하라.

### 💭 힌트

- 길이 검사: `len(s) in (4, 6)`
- 전부 숫자인지: `s.isdigit()` 한 번으로 판별
- 두 조건을 `and`로 결합해 한 줄로 가능

## 🟢 문제 2 — Valid Palindrome (회문 판별)

| 항목 | 내용 |
|------|------|
| 출처 | LeetCode |
| 번호 | 125 |
| 링크 | https://leetcode.com/problems/valid-palindrome/ |
| 카테고리 | 문자열, 회문, 투 포인터 |
| 난이도 | 🟢 Easy |

### 문제 요약

문자열 `s`에서 **영숫자(alphanumeric)만** 남기고 대소문자를 무시했을 때,
앞에서 읽으나 뒤에서 읽으나 같으면(회문, palindrome) `True`를 반환하라.

예: `"A man, a plan, a canal: Panama"` -> `True`

### 💭 힌트

- 전처리: `[c.lower() for c in s if c.isalnum()]` 로 영숫자만 추출 + 소문자화
- 방법 A) 슬라이싱: `cleaned == cleaned[::-1]`
- 방법 B) 투 포인터(two pointers): 양끝에서 좁혀오며 비교 -> O(1) 추가 공간
- 시간 복잡도: O(N)

## 🟡 문제 3 — Valid Anagram (애너그램 판별)

| 항목 | 내용 |
|------|------|
| 출처 | LeetCode |
| 번호 | 242 |
| 링크 | https://leetcode.com/problems/valid-anagram/ |
| 카테고리 | 문자열, 해시, Counter, 정렬 |
| 난이도 | 🟡 Easy~Medium |

### 문제 요약

두 문자열 `s`, `t`가 서로 **애너그램(anagram)** 인지 판별하라.
애너그램은 같은 문자를 같은 개수만큼 가지지만 순서만 다른 문자열이다.

예: `s = "anagram", t = "nagaram"` -> `True`

### 💭 힌트

- 방법 A) 정렬 비교: `sorted(s) == sorted(t)` -> O(N log N)
- 방법 B) 빈도 비교: `Counter(s) == Counter(t)` -> O(N)
- 길이가 다르면 즉시 `False` (조기 종료)

## 🟡 문제 4 — [1차] 다트 게임 (문자열 파싱 / Parsing)

| 항목 | 내용 |
|------|------|
| 출처 | 프로그래머스 (Programmers) — 2018 카카오 블라인드 1차 |
| 번호 | 17682 |
| 링크 | https://school.programmers.co.kr/learn/courses/30/lessons/17682 |
| 카테고리 | 문자열, 파싱, 상태 처리, 정규식(선택) |
| 난이도 | 🟡 Level 1 |

### 문제 요약

다트 3번의 결과를 담은 문자열을 파싱해 총점을 계산한다.
- 점수: 0~10 (두 자리 `10` 주의)
- 보너스: `S`(1제곱), `D`(2제곱), `T`(3제곱)
- 옵션: `*`(스타상 — 해당 점수와 **직전 점수**를 각각 2배), `#`(아차상 — 해당 점수 마이너스)

문자열 `dartResult`가 주어질 때 최종 점수를 반환하라.

### 💭 힌트

- 핵심 함정: 점수 `10`은 두 글자다. `replace("10", "k")` 로 한 글자 치환 후 다시 복원하는 트릭
- 또는 정규식 `(\d+)([SDT])([*#]?)` 로 (점수, 보너스, 옵션) 3원소를 한 번에 추출
- `*` 는 현재와 직전 점수를 모두 2배 -> 점수를 리스트로 보관하며 처리
- `#` 는 현재 점수만 음수로

## ⚫ 문제 5 — 신규 아이디 추천 (정규식 종합 / Kakao 기출)

| 항목 | 내용 |
|------|------|
| 출처 | 프로그래머스 (Programmers) — 2021 카카오 블라인드 |
| 번호 | 72410 |
| 링크 | https://school.programmers.co.kr/learn/courses/30/lessons/72410 |
| 카테고리 | 문자열, 정규식, 단계별 치환 |
| 난이도 | ⚫ Level 1 (기출, 단계가 많아 체감 난이도 높음) |

### 문제 요약

입력 아이디를 7단계로 순차 처리해 규칙에 맞는 추천 아이디를 만든다.

1. 모든 대문자를 소문자로
2. 알파벳 소문자, 숫자, `-`, `_`, `.` 외 문자 제거
3. 마침표 `.`가 2번 이상 연속되면 하나로 치환
4. 마침표가 처음이나 끝에 있으면 제거
5. 빈 문자열이면 `"a"` 대입
6. 길이가 16 이상이면 15자로 자르고, 끝이 `.`이면 제거
7. 길이가 2 이하면 마지막 글자를 길이 3이 될 때까지 반복해 붙임

### 💭 힌트

- 정규식 활용: 2단계 `re.sub(r"[^a-z0-9._-]", "", s)`, 3단계 `re.sub(r"\.{2,}", ".", s)`
- 4단계: `strip(".")` 로 양끝 마침표 제거
- 6단계: 슬라이싱 `s[:15]` 후 다시 `strip(".")`
- 7단계: `s + s[-1] * (3 - len(s))`
- 단계마다 결과를 다음 단계 입력으로 넘기는 **파이프라인** 구조
