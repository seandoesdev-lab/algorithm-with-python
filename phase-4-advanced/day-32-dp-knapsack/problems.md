## 1. Coin Change 🟢
- **출처:** [LeetCode #322](https://leetcode.com/problems/coin-change/)
- **요약:** 동전(무한 사용)으로 목표 금액을 만드는 **최소 동전 수**. 불가능하면 -1.
- **배낭 유형:** **무한 배낭**(각 동전을 무제한). 최적화 목표는 개수 최소화.
- **상태/점화식:** `dp[a] = min(dp[a], dp[a-c] + 1)`, `dp[0]=0`, 나머지 `INF`.
- **루프 방향:** 금액을 **정순**(무한이므로). Day 31의 복습이자 배낭 관점 재해석.

## 2. Coin Change II 🟡
- **출처:** [LeetCode #518](https://leetcode.com/problems/coin-change-ii/)
- **요약:** 동전(무한 사용)으로 목표 금액을 만드는 **조합의 수**(순서 무시).
- **배낭 유형:** **무한 배낭 + 카운팅**.
- **상태/점화식:** `dp[a] += dp[a-c]`, `dp[0]=1`.
- **함정:** **조합**을 세려면 **동전 루프가 바깥, 금액 루프가 안쪽**이어야 한다. 순서를 뒤집으면 순열을 세어 값이 커진다(대표적 실수).

## 3. Partition Equal Subset Sum 🟡
- **출처:** [LeetCode #416](https://leetcode.com/problems/partition-equal-subset-sum/)
- **요약:** 배열을 **합이 같은 두 부분집합**으로 나눌 수 있는가?
- **배낭 유형:** **0/1 부분집합 합**(가능여부). 전체 합 `S`가 홀수면 즉시 False, 아니면 `T=S//2`를 만들 수 있는지 판정.
- **상태/점화식:** `dp[s] = dp[s] or dp[s-x]`, `dp[0]=True`.
- **루프 방향:** 합 `s`를 **역순**(0/1이므로). 각 원소는 한 번만.

## 4. Target Sum 🔴
- **출처:** [LeetCode #494](https://leetcode.com/problems/target-sum/)
- **요약:** 각 수 앞에 `+` 또는 `-`를 붙여 합이 target이 되는 **방법의 수**.
- **배낭 유형:** **0/1 부분집합 합 + 카운팅**. `+`를 붙일 집합 합을 `P`, `-`를 붙일 집합 합을 `N`이라 하면 `P-N=target`, `P+N=S` → `P=(S+target)/2`.
- **핵심 변환:** "부호 배정 문제"를 "합이 `P`인 부분집합의 개수" 배낭으로 바꾼다.
- **함정:** `(S+target)`이 **홀수이거나 음수**면 답은 0(정수 부분집합 불가). `target`의 절댓값이 `S`보다 크면 0. 변환 전에 반드시 예외 처리.

## 5. Ones and Zeroes 🔴
- **출처:** [LeetCode #474](https://leetcode.com/problems/ones-and-zeroes/)
- **요약:** 이진 문자열 배열에서 **0을 최대 m개, 1을 최대 n개** 쓰며 고를 수 있는 **최대 부분집합 크기**.
- **배낭 유형:** **2차원 0/1 배낭**(용량 축이 2개: 0의 개수, 1의 개수).
- **상태/점화식:** `dp[i][j] = max(dp[i][j], dp[i-zeros][j-ones] + 1)`.
- **루프 방향:** 각 문자열마다 `i`(0개수)와 `j`(1개수)를 **둘 다 역순**으로. 그래야 각 문자열이 한 번만 쓰인다.

## 6. 도둑질 ⚫ (기출)
- **출처:** [프로그래머스 #42897](https://school.programmers.co.kr/learn/courses/30/lessons/42897)
- **요약:** 집들이 **원형**으로 배치. 인접한 두 집은 동시에 못 턴다. 훔칠 수 있는 최대 금액.
- **유형:** 배낭이 아니라 **선택 DP(House Robber)** 의 원형 변형이지만, "택하거나 건너뛰거나"라는 배낭적 사고가 그대로 쓰인다.
- **핵심:** 원형이라 **첫 집과 마지막 집을 동시에 털 수 없다.** 따라서 두 경우로 나눠 각각 일자형 House Robber를 돌리고 max를 취한다:
  - (A) 0번 집 포함 후보 → `money[0 : n-1]` 구간
  - (B) 마지막 집 포함 후보 → `money[1 : n]` 구간
- **점화식:** `dp[i] = max(dp[i-1], dp[i-2] + money[i])` ([[day-31-dp/concept|Day 31 House Robber]]와 동일).

## 추천 풀이 순서
1. **1 → 2** : 무한 배낭의 최소화(개수)와 카운팅(조합). "루프 순서 = 조합/순열"을 체득.
2. **3 → 4** : 0/1 부분집합 합(가능여부 → 카운팅 + 변환). Target Sum의 `(S+target)/2` 변환이 핵심.
3. **5** : 용량 축을 2개로 늘리는 다차원 0/1 배낭.
4. **6** : 기출 — 원형 배열을 "경계 두 번" 트릭으로 일자형으로 환원.
