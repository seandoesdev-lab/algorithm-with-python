풀이 순서는 **BFS(가중치 1) 대비 → 표준 다익스트라 → 기출 → minimax 변형 → 제약 최단 경로**로,

## 1. Shortest Path in Binary Matrix 🟢
- **출처:** [LeetCode #1091](https://leetcode.com/problems/shortest-path-in-binary-matrix/)
- **유형:** BFS (모든 간선 가중치 1) — 다익스트라가 **필요 없는** 대비 사례
- **문제:** `n x n` 이진 격자에서 값이 0인 칸만 밟아 좌상단 `(0,0)`에서 우하단 `(n-1,n-1)`까지 가는 최단 경로 길이(방문한 칸 수)를 구하라. **8방향** 이동 가능. 경로가 없으면 `-1`.
- **입력/출력:** `grid: List[List[int]]` → `int`
- **제약:** `1 <= n <= 100`, 각 칸은 0 또는 1
- **핵심 힌트:**
  - 모든 이동 비용이 동일(=1)하므로 **BFS 한 번**이면 최단이다. 힙(다익스트라)은 오버킬.
  - 시작·도착 칸이 1이면 즉시 `-1`.
  - 8방향 델타 `(-1,-1)...(1,1)` 8개. 방문 처리는 큐에 넣는 **순간**에.

## 2. Network Delay Time 🟡
- **출처:** [LeetCode #743](https://leetcode.com/problems/network-delay-time/)
- **유형:** 표준 단일 출발점 다익스트라
- **문제:** 방향 그래프의 간선 `times[i] = [u, v, w]`(u→v 전파 시간 w). 노드 `k`에서 신호를 보낼 때 **모든 노드가 신호를 받는 최소 시간**을 구하라. 일부라도 못 받으면 `-1`.
- **입력/출력:** `times: List[List[int]], n: int, k: int` → `int`
- **제약:** `1 <= k <= n <= 100`, `1 <= times.length <= 6000`, `1 <= w <= 100`
- **핵심 힌트:**
  - `k`에서 다익스트라 → `dist[]`. 답은 **`max(dist[1..n])`** (가장 늦게 받는 노드).
  - `max`가 INF면 도달 못 한 노드가 있다는 뜻 → `-1`.
  - "모든 노드까지 최단" + "그중 최댓값"이라는 2단 구조가 이 문제의 포인트.

## 3. 배달 ⚫ (기출)
- **출처:** [프로그래머스 #12978](https://school.programmers.co.kr/learn/courses/30/lessons/12978)
- **유형:** 무방향 가중 그래프 다익스트라
- **문제:** 1번 마을에 있는 음식점이 각 마을로 배달한다. 마을 수 `N`, 도로 `road[i] = [a, b, c]`(a↔b **양방향**, 시간 c), 시간 `K`가 주어질 때, **1번에서 K 이하로 배달 가능한 마을 수**를 구하라(1번 포함).
- **입력/출력:** `N: int, road: List[List[int]], K: int` → `int`
- **제약:** `1 <= N <= 50`, `1 <= road.length <= 2000`, `1 <= c <= 10000`, 두 마을 사이 여러 도로 가능
- **핵심 힌트:**
  - **양방향**이므로 `graph[a]`와 `graph[b]` **둘 다** 간선을 넣는다(한쪽만 넣으면 조용히 오답).
  - 1번에서 다익스트라 후 `dist[i] <= K`인 마을 수를 센다.
  - 두 마을 사이 도로가 여러 개여도 완화가 알아서 최솟값을 고른다.

## 4. Path With Minimum Effort 🟡
- **출처:** [LeetCode #1631](https://leetcode.com/problems/path-with-minimum-effort/)
- **유형:** minimax 격자 다익스트라 (합이 아니라 "최대 간선" 최소화)
- **문제:** `heights[r][c]` 격자에서 좌상단→우하단으로 4방향 이동. 한 경로의 "**effort**"는 경로상 **인접 칸 높이차의 최댓값**이다. 모든 경로 중 effort가 **최소**인 값을 구하라.
- **입력/출력:** `heights: List[List[int]]` → `int`
- **제약:** `1 <= rows, cols <= 100`, `1 <= heights[i][j] <= 10^6`
- **핵심 힌트:**
  - 다익스트라 골격 그대로, 완화만 `nd = max(cur, abs(높이차))`로 바꾼다.
  - `effort[r][c]` = "그 칸까지의 최소 effort". 도착 칸을 힙에서 꺼내는 순간이 답.
  - 대안: **이분 탐색 + BFS**("effort ≤ x로 도달 가능?"을 x에 대해 이분) — [[day-18-binary-search/concept|Day 18]] 응용.

## 5. Cheapest Flights Within K Stops 🔴
- **출처:** [LeetCode #787](https://leetcode.com/problems/cheapest-flights-within-k-stops/)
- **유형:** **제약(경유 수 ≤ K)** 이 붙은 최단 경로 — 순수 다익스트라로 안 됨
- **문제:** 도시 `n`개, 항공편 `flights[i] = [u, v, w]`(u→v 요금 w). `src`에서 `dst`까지 **최대 K번 경유(=간선 K+1개 이하)** 하는 **최소 요금**을 구하라. 불가능하면 `-1`.
- **입력/출력:** `n: int, flights: List[List[int]], src: int, dst: int, k: int` → `int`
- **제약:** `1 <= n <= 100`, `0 <= flights.length <= n*(n-1)`, `1 <= w <= 10^4`, `0 <= k < n`
- **핵심 힌트:**
  - 순수 다익스트라는 요금만 최소화하고 **경유 수 제약을 못 건다** → 상태 확장 필요.
  - **정석: 벨만-포드식 K+1회 완화.** 매 라운드 직전 `dist`의 스냅샷으로만 갱신해 "한 라운드에 간선 1개씩만" 늘어나게 한다.
  - 대안: `(비용, 도시, 남은 경유 수)`를 힙에 넣는 상태 다익스트라.

## 6. Swim in Rising Water ⚫ (기출)
- **출처:** [LeetCode #778](https://leetcode.com/problems/swim-in-rising-water/)
- **유형:** minimax 다익스트라 (경로상 최대 고도 최소화)
- **문제:** `grid[r][c]` = 그 칸의 고도. 시각 t에 고도 ≤ t인 칸은 물에 잠긴다. `(0,0)`에서 `(n-1,n-1)`까지 4방향으로 헤엄쳐 갈 수 있는 **가장 이른 시각 t**를 구하라(경로상 최대 고도 = 답).
- **입력/출력:** `grid: List[List[int]]` → `int`
- **제약:** `1 <= n <= 50`, `grid`는 `0..n*n-1`의 순열
- **핵심 힌트:**
  - "경로상 **최대 고도**를 최소화" = minimax. 완화는 `nt = max(cur_t, grid[nr][nc])`.
  - 힙에 `(현재까지_최대고도, r, c)`. 도착 칸을 꺼내는 순간이 답.
  - 대안: 이분 탐색 + BFS, 또는 [[day-36-union-find/concept|Union-Find]]로 고도 오름차순 병합(추후 학습).

- [ ] `heapq` (거리, 정점) 튜플 순서와 지연 삭제(`if d > dist[u]: continue`)를 손에 익힌다

- [ ] minimax 변형은 완화 식만 `+` → `max`로 바꾼다 (4·6번)
- [ ] 제약(K 경유)이 붙으면 상태를 확장하거나 벨만-포드식 완화를 쓴다 (5번)
- [ ] 도달 불가(INF)를 `-1` 등으로 변환하는 걸 잊지 않는다 (2·5번)
